# MVP Checklist

Use this checklist to track the minimum scope for the first meaningful release.

## Scope Framing

- [ ] Define the target audience for the initial release
- [ ] Define the core user value the MVP must deliver
- [ ] Define what is explicitly out of scope for the MVP

## Product Readiness

- [ ] Confirm the main game flow is stable
- [ ] Confirm the initial difficulty experience is acceptable
- [ ] Confirm settings and consent flows are acceptable for first release
- [ ] Confirm feedback collection is ready for real users

## Deployment Readiness

- [ ] Confirm the target deployment environment
- [ ] Confirm the domain and HTTPS plan
- [ ] Confirm the basic server/runtime configuration
- [ ] Confirm the initial deploy and smoke-test process

## Validation

- [ ] Define what success looks like for the MVP
- [ ] Define what data or feedback will be collected
- [ ] Define the first follow-up decisions that MVP feedback should inform
